## 1.1 Python ဆိုတာ ဘာလဲ?

- Python ကို **Guido van Rossum** ဆိုတဲ့ ဒတ္တိယဆရာကြီးက ဖန်တီးခဲ့ပြီး လူကြိုက်များလို့ **Benevolent Dictator For Life (BDFL)** လို့ ခေါ်ကြတယ်။
- **1991** ခုနှစ်မှာ ပထမဆုံး Release လုပ်ခဲ့တယ်။
- Python Interpreter ကို **Windows, Linux, Mac OS** ဆိုတဲ့ Operating System အများအပြားမှာ Run လို့ရတယ်။
- Python ကို သုံးသူတွေကို **Pythonists** သို့ **Pythonistas** လို့ ခေါ်ကြတယ်။

---

## 1.2 Python ဘာကြောင့် လှိုင်းကြောင်းကြီးလို လူကြိုက်များလဲ?

### (a) အလကားသုံးလို့ရတယ် (Free)
- Python ကို **အခမဲ့ သုံးလို့ရတယ်**။
- Open-source Community ကြီးကြီးမားမားက ထောက်ပံ့ပေးထားတယ်။
- Platform အများအပြားမှာ Python Interpreter ရရှိနိုင်တယ်။

### (b) Software Quality (အရည်အသွေးကောင်း)
- Python က Traditional scripting languages တွေထက် **ရှင်းလင်းတဲ့ code** ရေးလို့ရတယ်။
- Code အရမ်း **Readable** ဖြစ်လို့ ပြန်ပြင်လို့လည်း လွယ်ကူတယ်။
- Modules, Packages, OOP စတဲ့ **Reusable mechanism** တွေက မိုက်မိုက်ကောင်းကောင်း support ပြုတယ်။

### (c) Developer Productivity (Developer တွေအတွက် output မြင့်)
- Statically typed languages တွေနဲ့ compared လုပ်ရင် Python က **အလုပ် output မြင့်** တယ်။
- Code ရေးရတာ **တိုတယ်၊ လွယ်တယ်**။
- Debugging နဲ့ Maintenance ကလည်း လွယ်ကူတယ်။
- **Compile လုပ်စရာမလိုဘူး** — ရေးပြီးသွားတာနဲ့ Run လို့ရတယ်။

### (d) Program Portability (Platform တစ်ခုကနေ တစ်ခုသို့ လွယ်ကူစွာပြောင်းသုံးနိုင်)
- Python Program တွေက **OS အများစုမှာ ပြန်မပြင်ဘဲ Run** လို့ရတယ်။
- GUI, Database, Web, OS Interface တို့အပါအဝင် Library အမျိုးအစားအားလုံးက Platform အတော်များများမှာ Run လို့ရတယ်။

### (e) Support Libraries (Library Support အလွန်ကောင်း)
- Standard Library မှာ **Networking, Text Processing, File Handling** အစရှိသဖြင့်အရာရာပါပြီးသား။
- အပြင်မှာလည်း **Third-party Libraries** များများစွာရှိတယ် —  
  ML, Web, Game, Science, AI အထိ။

### (f) Component Integration (အခြား System တွေနဲ့ ချိတ်ဆက်သုံးလို့ရ)
- Python က **C, C++, Java** Libraries တွေကို ချိတ်ပြီး သုံးလို့ရတယ်။
- COM, .NET Framework နဲ့ပဲ အလုပ်လုပ်လို့ရတယ်။
- SOAP, XML-RPC, CORBA တို့လို Network Interface မျိုးစုံလည်း Interact လို့ရတယ်။

### (g) Enjoyment (သုံးရတာပျော်စရာကောင်း)
- Python ကို သက်တောင့်သက်သာ သုံးလို့ရတယ်။
- Built-in toolset အပြည့်အစုံရှိတယ်။
- Programming ကို **အလုပ်မဟုတ်ပဲ ပျော်ရွှင်စရာ** တစ်ခုလို ခံစားစေတယ်။

## 1.6 Programming Paradigms

### 1.6.1 Programming Paradigm ဆိုတာဘာလဲ?

- Paradigm ဆိုတာ Program အား ဖွဲ့စည်းရေးရာ **ပုံစံ** ပါ။
- Functional, Procedural, OOP, Event-driven စတဲ့ Programming Model များရှိတယ်။
- Python က Language တစ်ခုတည်းနဲ့ **Paradigm မျိုးစုံ** Support ကြောင့် Flexible တယ်။

---

## 1.6.2 Functional Programming Model

- Functional Programming က Problem ကို **Function**များအဖြစ် ခွဲထုတ်ပြုလုပ်တယ်။
- Function တစ်ခုက **Input → Output** ပြန်ပေးတယ်။
- map(), reduce(), filter(), lambda လို FP tools တွေကို Python support လုပ်တယ်။

### ဥပမာ – Factorial, Fibonacci
```text 
factorial(n) = 1 if n == 0
= n * factorial(n - 1)

fibo(n) = 0 if n == 0
= 1 if n == 1
= fibo(n-2) + fibo(n-1)
```

### Functional Programming Properties
- Output က Input မှာပဲ မူတည်တယ်  
- Same input → same output  
- No side effects  
- Parallel execution အတွက် သင့်တင့်တယ်  
- Declarative style — “ဘာလိုုချင်လဲ” ကိုသာပြောတယ်  

---

## 1.6.3 Procedural Programming Model

- Statement (Procedure) တွေကို **အဆင့်လိုက်** ဆောင်ရွက်ကာ Program state ကိုပြောင်းလဲတယ်
- Functions တွေက pure math functions မဟုတ်ဘူး (state modify လုပ်နိုင်)

### Procedural Properties
- Statements က value မထုတ်ပေးဘူး → program state ပြောင်းလဲပေးတယ်  
- Same expression က state မတူရင် result မတူနိုင်တယ်  
- Global state ပြောင်းလဲနိုင်တယ်  
- Imperative style — “ဘယ်လိုလုပ်မယ်” ကိုတိတိကျကျပြကာ code ရေးတယ်  

---

## 1.6.4 Object-oriented Programming (OOP) Model

- Real-world objects (Student, Course, Exam, Staff…) ကို Computer ထဲမှာ Object အဖြစ် တည်ဆောက်တယ်

### Object အကြောင်းအရာများ
- **State** → Attributes  
- **Behavior** → Methods  
- Object တွေဟာ Class မှ ထွက်လာသူတွေ  
- Object တွေ အပြန်အလှန် message ပို့၍ method ခေါ်ပြီး အလုပ်လုပ်တယ်  

---

## 1.6.5 Event-driven Programming Model

- GUI (Button, Menu, Scrollbars) တို့မှာ တွေ့နိုင်တဲ့ Programming Style
- User က Click/Scroll လုပ်မှ Event ဖြစ်တယ်
- Listener Function တွေက Event ဖြစ်လိုက်တာနဲ့ ပြန်လုပ်ဆောင်တယ်

### Properties
- Event order ညှာတာမရှိ  
- User interaction ပေါ်မူတည်  
- Listener တွေ asynchronous ဖြစ်ရမယ်  
- GUI, Web App, Games တို့မှာ အများဆုံးသုံး
